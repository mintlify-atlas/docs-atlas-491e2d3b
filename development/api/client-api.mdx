## Overview

Client API commands handle game launching, build management, mod installation, and data persistence. These commands are split across three files:
- `src-tauri/src/app/gui/commands/client.rs` - Client launching and API interactions
- `src-tauri/src/app/gui/commands/data.rs` - Data management
- `src-tauri/src/app/gui/commands/system.rs` - System information

## Client Launching

### run_client

Launch the Minecraft client with LiquidBounce. This command handles the entire launch process including downloading assets, libraries, and starting the game.

<CodeGroup>
```rust Signature
#[tauri::command]
pub(crate) async fn run_client(
    client: Client,
    build_id: u32,
    options: Options,
    mods: Vec<LoaderMod>,
    window: Window,
    app_state: tauri::State<'_, AppState>,
) -> Result<(), String>
```

```javascript Usage
import { invoke } from '@tauri-apps/api/core';
import { listen } from '@tauri-apps/api/event';

// Listen for progress updates
await listen('progress-update', (event) => {
  console.log('Progress:', event.payload);
});

// Listen for process output
await listen('process-output', (event) => {
  console.log('Output:', event.payload);
});

// Launch the client
await invoke('run_client', {
  client: clientInstance,
  buildId: 12345,
  options: launcherOptions,
  mods: selectedMods
});
```
</CodeGroup>

<ParamField path="client" type="Client" required>
  The LiquidBounce API client instance
</ParamField>

<ParamField path="build_id" type="number" required>
  The build ID to launch (obtained from `request_builds`)
</ParamField>

<ParamField path="options" type="Options" required>
  Launcher options including account, memory, JVM args, etc.
  <Expandable title="Options structure">
    <ParamField path="start.account" type="MinecraftAccount" required>
      The Minecraft account to use for launching
    </ParamField>
    <ParamField path="start.memory" type="number" default="4096">
      Memory allocation in MB
    </ParamField>
    <ParamField path="start.javaDistribution" type="string">
      Java distribution to use (e.g., "Zulu", "Adoptium")
    </ParamField>
    <ParamField path="start.jvmArgs" type="string[]">
      Additional JVM arguments
    </ParamField>
    <ParamField path="start.customDataPath" type="string">
      Custom data directory path (empty for default)
    </ParamField>
    <ParamField path="launcher.keepLauncherOpen" type="boolean">
      Whether to keep launcher open after starting the game
    </ParamField>
    <ParamField path="launcher.concurrentDownloads" type="number">
      Number of concurrent downloads
    </ParamField>
    <ParamField path="premium.account" type="ClientAccount">
      LiquidBounce account for premium features
    </ParamField>
    <ParamField path="premium.skipAdvertisement" type="boolean">
      Skip advertisements (requires premium)
    </ParamField>
  </Expandable>
</ParamField>

<ParamField path="mods" type="LoaderMod[]" required>
  Array of mods to load with the client
</ParamField>

<ParamField path="window" type="Window" required>
  Tauri window instance (automatically passed)
</ParamField>

<ParamField path="app_state" type="AppState" required>
  Application state (automatically passed)
</ParamField>

<ResponseField name="result" type="void">
  Returns nothing on successful launch start. The game runs in a separate thread.
</ResponseField>

**Events Emitted:**

<ResponseField name="progress-update" type="ProgressUpdate">
  Emitted during launch preparation with progress information
  <Expandable title="types">
    - `SetLabel` - Status message string
    - `SetProgress` - Progress value (0-100)
    - `SetMax` - Maximum progress value
  </Expandable>
</ResponseField>

<ResponseField name="process-output" type="string">
  Game console output (stdout and stderr)
</ResponseField>

<ResponseField name="client-error" type="void">
  Emitted when the client crashes or fails to start
</ResponseField>

<ResponseField name="client-exited" type="void">
  Emitted when the client exits normally
</ResponseField>

<Warning>
  Returns an error if the client is already running: `"client is already running"`
</Warning>

### request_builds

Retrieve builds for a specific branch.

<CodeGroup>
```rust Signature
#[tauri::command]
pub(crate) async fn request_builds(
    client: Client,
    branch: &str,
    release: bool
) -> Result<Vec<Build>, String>
```

```javascript Usage
// Get all builds (including nightlies)
const builds = await invoke('request_builds', {
  client: clientInstance,
  branch: 'nextgen',
  release: false
});

// Get only release builds
const releases = await invoke('request_builds', {
  client: clientInstance,
  branch: 'nextgen',
  release: true
});
```
</CodeGroup>

</ParamField>

<ParamField path="branch" type="string" required>
